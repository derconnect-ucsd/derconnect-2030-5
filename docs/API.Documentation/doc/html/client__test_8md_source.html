1 Client Test Application
2 =======================
3 
4 Usage
5 -----
6 
7 The `client_test` application is a command line tool with the following
8 arguments:
9 
10  client_test interface [device certs..] <subtype[:1] | URI> [commands]
11 
12 The `interface` argument is the the name of the network interface,
13 running the `client_test` application with no arguments will list the
14 available network interfaces.
15 
16 The `device` argument names the device certificate and private key pair
17 used for TLS, the application will load the certificate `device.x509`
18 and use the private key file `device.pem`.
19 
20 The `certs` argument is a list of CA certificates or certificate
21 directories to be loaded. If a directory is specified, then all the
22 certificates within the directory are loaded. If no certficates or
23 directories then the certificates from `./certs` in the local directory
24 are loaded.
25 
26 The `<subtype[:1] | URI>` argument delimits the list of certificates and
27 specifies either to perform xmDNS service discovery, or to retrieve the
28 document at the specified URI.
29 
30 Subtype DNS-SD queries
31 ----------------------
32 
33 The `subtype` can be one of:
34 
35 - `bill` - to discover Billing servers
36 - `dr` - to discover Demand Response / Load Control servers
37 - `derp` - to discover Distributed Energy Resources servers
38 - `file` - to discover File Download servers
39 - `msg` - to discover Messaging servers
40 - `ppy` - to discover Prepayment servers
41 - `rsps` - to discover Response servers
42 - `tp` - to discover Pricing servers
43 - `tm` - to disocver Time servers
44 - `upt` - to discover Metering servers
45 - `edev` - to discover End Device servers
46 - `mup` - to discover Meter Mirroring servers
47 - `sdev` - to discover Self Device servers
48 - `smartenergy` - to discover IEEE 2030.5 servers of any type
49 
50 When the number `1` is affixed to the `subtype`, the `client_test` sets
51 the `QU` bit to `1` in the service discovery request indicating that a
52 unicast response is desired.
53 
54 Since the `subtype` argument can delimit the list of certifcates it
55 follows that there can not be a certificate or certificate directory
56 with the same name as the `subtype` argument.
57 
58 After the service discovery is performed the `client_test` application
59 will retreive the DeviceCapability resource at the location given by the
60 `dcap` key in discovery response.
61 
62 URI retrieval
63 -------------
64 
65 If a `subtype` is not specified in the list of arguments, the
66 `client_test` application attempts to parse a URI. The first argument
67 that matches an absolute form URI (e.g <https:://>\[::1\]/dcap ) will
68 delimit the list of certificates and attempt to retrieve the URI. The
69 URI scheme specified determines the type of connection used to retreive
70 the resource either `http` for an unencrypted connection or `https` for
71 a secure TLS connection. If no port is specified, the default ports for
72 HTTP and HTTPS are used, 80 and 443 respectively.
73 
74 Commands
75 --------
76 
77 The last set of arguments are a list of commands to be interpreted.
78 These can be any of the following:
79 
80 - `sfdi SFDI` - Use the specified sfdi in performing device registration.
81 This command should be the first command in the list of commands.
82 
83 - `register` - Register the client EndDevice with the server. The
84 `client_test` application will first check the available EndDevices in
85 the EndDeviceList resource. If an EndDevice with a matching sFDI is not
86 found, the client POSTs an EndDevice instance that contains the sFDI
87 computed from the device certificate. If the POST succeeds or the
88 EndDevice with a matching sFDI was found, the client will retreive the
89 Registration resource linked from the EndDevice instance, the pIN value
90 should be 111115, if not the registration fails. If registritation
91 succeeds and the EndDevice has an associated DER instance, then PUT the
92 device settings to the links provided in the DER instance.
93 
94 - `primary` - Perform all the functions of `register`, then retreive the
95 FunctionSetAssignmentsList associated with each EndDevice. Determine the
96 FunctionSetAssignments with the highest priority according to the list
97 ordering and retrieve only the DERProgramList associated with that FSA,
98 then retrieve the DERCurveList and DERControlList associated with the
99 highest priority DERProgram in the DERProgramList. Finally schedule the
100 events when retrieval is complete.
101 
102 - `all` - Perform the same functions as `primary`, except all DERPrograms
103 and the associated DERCurveLists and DERControlLists are retrieved.
104 
105 - `poll interval` - Set the poll rate for active events in seconds, the
106 default is 300 seconds or 5 minutes.
107 
108 - `load sfdi directory` - Load the device settings located in `directory`
109 for the EndDevice with the `sfdi` specified.
