Detailed Description

A Connection extends a TcpPort and provides an interface to either a TCP or TLS connection. Connections are established by calling either conn_connect or conn_accept, the "secure" parameter determines whether to use TLS or not. After a connection has been initiated the client (or server) calls conn_session to check the status of the connection. In the case of TLS, repeat calls to conn_session might be required to fully negotiate a TLS session. After a session has been established, use conn_read and conn_write to read and write data to the connection stream.
